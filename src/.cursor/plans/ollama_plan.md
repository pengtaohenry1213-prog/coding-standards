# 经济模型系统 v2 升级项目立项书

## 1. 项目基本信息

| 项目名称 | 类型 | 决策人 |
|---------|------|--------|
| 经济模型系统 v2 升级 | 技术升级 + 功能增强 | PMO + 技术负责人 |

## 2. 项目背景与目标

### 现状问题

当前经济模型系统 v1 存在以下局限性：

- 架构陈旧：自研公式引擎 + 单人离线编辑模式
- 功能单一：不支持多人实时协作
- 扩展性差：无法平滑升级到现代化架构
- 安全合规：现有架构难以满足等保要求

### 升级目标

1. 构建新一代在线协作平台，实现从单人离线到多人实时协作的升级
2. 引入专业 HyperFormula 引擎，确保财务公式计算 100% 准确
3. 实现内网可部署和完整操作审计，满足等保要求
4. 打造可扩展的技术架构，支持后续 AI 智能助手等功能

## 3. 范围定义

### In Scope（包含）

- P0: 表格编辑 + 公式引擎 + 版本管理 + 数据导入导出
- P1: 树形表格 + 数据格式化 + 权限控制
- P2: 后续阶段包含 Yjs 协作 + Supabase 后端 + RAG 智能助手

### Out of Scope（不包含）

- 第三方依赖库开发
- 老旧 v1 功能模块迁移
- 与外部系统的深度集成

## 4. 验收标准

### 功能验收清单

| 序号 | 功能项 | 验收条件 |
|-----|-------|---------|
| 1 | 单元格双击编辑 | 可编辑任意单元格 |
| 2 | 公式计算 | SUM/XIRR/XNPV/NPV/IRR 计算正确 |
| 3 | 版本管理 | 草稿→已提交→已锁定流程正常 |
| 4 | Excel 导入导出 | 模板格式兼容 |
| 5 | 权限控制 | 锁定版本不可编辑 |

### 性能指标

- 表格加载 < 2s (1000 行数据)
- 公式计算 < 100ms (100 个公式)
- 虚拟滚动：滚动流畅无卡顿

### 安全合规要求

- 完整操作审计日志
- 敏感数据脱敏
- 权限矩阵生效

## 立项书创建计划

基于已收集的 v2 项目资料，创建正式的立项书文档。

### 文档结构

创建 `docs/requirements/step-01-v2-project-proposal.md`，包含以下章节：

1. **项目基本信息**
  - 项目名称：经济模型系统 v2 升级
  - 项目类型：技术升级 + 功能增强
  - 决策人：PMO + 技术负责人
2. **项目背景与目标**
  - 现状问题（v1 的三大局限）
  - 升级目标（5 项核心目标）
3. **范围定义（In Scope / Out of Scope）**
  - In Scope: P0 核心功能 + P1 增强功能
  - Out of Scope: P2 扩展功能（Yjs 协作、Supabase、RAG）
4. **验收标准**
  - 功能验收清单（8 项）
  - 性能指标（3 项）
  - 安全合规要求（3 项）
5. **里程碑计划**
  - M0: 项目初始化（1天）
  - M1-M5: 详细里程碑
6. **风险评估**
  - P0 风险：HyperFormula 兼容性、Yjs 复杂度、Excel 导入异常
  - 应对措施
7. **Human Gate 1 决策表**
  - PMO 检查项
  - Security 检查项
  - 决策：PASS / CONDITIONAL / REJECT

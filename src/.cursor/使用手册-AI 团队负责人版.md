# AI 团队负责人使用手册

## 🎯 角色定位

```
你 = 🧠 AI 团队负责人
职责 = 只写 step，不写代码
工具 = Multi-Agent 自动执行 + Human Gate 双审
```

---

## 🔄 核心流程图

```
┌─────────────────────────────────────────────────────────────┐
│  1️⃣ 你（AI 团队负责人）                                      │
│  编写 doc/steps/stepN.md                                     │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  2️⃣ 🔴 Human Gate 1（执行前）                               │
│  PMO + Security 评审                                        │
│  安全扫描 → 权限校验 → 质量检查                                │
│  结果：PASSED / REJECTED / WARNING                          │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  3️⃣ AI Agent 执行                                          │
│  Planner → Frontend → Backend → Test → Reviewer             │
│  每个 todo 独立 commit，可并行                                 │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  4️⃣ 🟢 Human Gate 2（执行后）                               │
│  PMO + Security 复审                                        │
│  结果校验 → 日志校验 → 可追溯性                                 │
│  结果：ACCEPTED / ROLLBACK                                  │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  5️⃣ 验收通过 → Git 提交                                     │
└─────────────────────────────────────────────────────────────┘
```

---

## 📁 文件位置

| 类型 | 位置 |
|------|------|
| Step 文件 | `doc/steps/stepN.md` |
| Plan 文件 | `.cursor/plans/stepN-plan.md` |
| 测试报告 | 写入 Plan 文件的 `## 🧪 测试报告` |

---

## 🚀 执行命令

### 全自动模式

```bash
/run-all step1
```

### 调试模式（单步执行）

```bash
/planner step1    # 生成 Plan
/frontend step1   # 前端开发
/backend step1    # 后端开发
/test step1       # 测试验证
/reviewer step1   # 代码审查
```

---

## 📋 Step 文件模板

```markdown
## 1. Step 编号
Stage1-Step01

## 2. Step 目标
当前 step 要完成的具体任务

## 3. 输入条件
前置资源/数据/环境

## 4. 执行动作
具体要做的操作

## 5. 输出物 & 验收标准
产出内容 + 验证方式

## 6. 关联依赖
前置 step / 后续 step
```

---

## ⚠️ 强制规则

| 规则 | 说明 |
|------|------|
| 不允许跳步骤 | 必须按顺序执行 |
| Human Gate 双审 | 执行前 + 执行后必须审查 |
| 测试必须实测 | 必须有实际命令和输出 |
| 每个 todo 独立 commit | type + step + todo-id |
| Plan 必须写入 `.cursor/plans/` | - |

---

## 🔧 失败处理

```
Review 不通过 → Fix Agent 修复 → 重新 Review → 闭环
```

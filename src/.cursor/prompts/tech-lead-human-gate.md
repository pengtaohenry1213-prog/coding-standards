# Human Gate：Tech Lead 技术评审

本 prompt 负责对技术方案进行评审。遵循 CO-STAR 框架的 Mode B 格式。

---

## 1. Context（背景）

Tech Lead Human Gate 是技术把关关卡，在技术方案设计、架构变更、重大重构前对技术方案进行评审。

### 触发时机

| 时机 | 说明 |
|------|------|
| 技术方案设计阶段 | 方案实施前评审 |
| 架构变更时 | 架构调整前评审 |
| 重大重构前 | 重构实施前评审 |
| 技术选型决策时 | 新技术引入前评审 |

---

## 2. Objective（审核目标）

确保技术方案可行、代码质量合格、性能与安全达标、运维与回滚方案完善。

---

## 3. Checklist（审核清单）

### 1. 技术方案

- [ ] 方案可行，有实现路径
- [ ] 符合现有架构设计原则
- [ ] 扩展性考虑周全
- [ ] 与系统其他部分解耦或明确耦合点

### 2. 代码质量

- [ ] 符合项目编码规范（coding-standards.md）
- [ ] 有单元测试覆盖
- [ ] 代码可维护、可读性好

### 3. 性能与安全

- [ ] 性能影响评估（时间/空间复杂度）
- [ ] 安全风险识别（参考 security-rules.md）
- [ ] 有性能基准或压测计划

### 4. 运维与回滚

- [ ] 有回滚方案
- [ ] 部署步骤清晰
- [ ] 监控/告警到位
- [ ] 数据迁移方案（如涉及）

### 5. 团队协作

- [ ] 技术方案已评审（RFC/设计文档）
- [ ] 相关方已通知（后端/前端/运维/DBA）
- [ ] 知识转移计划（如需要）

---

## 4. Decision（决策格式）

| 决策 | 条件 |
|------|------|
| **PASS** | 所有检查项通过 |
| **CONDITIONAL** | 部分检查项待完善，可有条件执行 |
| **REJECT** | 存在架构/安全/性能风险，需重新设计 |

### 评审维度权重

| 维度 | 权重 | 说明 |
|------|------|------|
| 架构合理性 | 30% | 是否符合长期技术方向 |
| 安全性 | 25% | 是否有安全隐患 |
| 可行性 | 20% | 团队能力、时间是否可行 |
| 性能影响 | 15% | 对系统性能的影响 |
| 维护成本 | 10% | 后续维护复杂度 |

---

## 5. Decision Record（决策记录）

```markdown
## Human Gate 决策

| 项目 | 内容 |
|------|------|
| Gate 类型 | Tech Lead |
| 变更类型 | 新功能 / 重构 / 技术选型 / 架构调整 |
| 涉及系统 | ... |
| 决策 | PASS / CONDITIONAL / REJECT |
| 风险描述 | ... |
| 回滚方案 | ... |
| Tech Lead | ___ |
```

---

## 6. Closure（闭环要求）

- REJECT 必须重新设计方案
- PASS 后方可进入开发
- CONDITIONAL 需在 Gate 2（产出评审）前完成整改

---

## 7. Prohibitions（禁止事项）

- 未评审直接实现
- 绕过安全检查
- 无回滚方案的破坏性变更

# Human Gate：Test 测试验收

本 prompt 负责对测试报告进行验收。遵循 CO-STAR 框架的 Mode B 格式。

---

## 1. Context（背景）

Test Human Gate 是测试质量把关关卡，在测试报告提交、发布前质量评审、缺陷关闭前复验时对测试结果进行评审。

### 触发时机

| 时机 | 说明 |
|------|------|
| 测试报告提交时 | 测试完成后评审 |
| 发布前质量评审时 | 发布前最终确认 |
| 缺陷关闭前复验时 | 缺陷修复后复验 |

---

## 2. Objective（审核目标）

确保测试用例覆盖完整、缺陷管理规范、报告质量达标、回归与冒烟测试通过。

---

## 3. Checklist（审核清单）

### 1. 用例覆盖

- [ ] **核心场景用例 100%** 覆盖
- [ ] **非核心场景 ≥95%** 覆盖
- [ ] 含正向、逆向、边界、异常场景
- [ ] 覆盖目标与发布范围一致

### 2. 缺陷管理

- [ ] 缺陷分级明确（P0/P1/P2/P3）
- [ ] 缺陷可复现（步骤清晰）
- [ ] 缺陷有证据（截图/日志摘要）
- [ ] P0/P1 缺陷已全部修复或已有应对方案

### 3. 报告质量

- [ ] 包含范围与环境说明
- [ ] 通过率摘要清晰
- [ ] **未通过项列表**：级别/影响/建议
- [ ] 发布建议明确

### 4. 回归与冒烟

- [ ] 回归覆盖修改点 + 合理关联面
- [ ] 冒烟验证主路径通过

### 优先级定义

| 级别 | 定义 | 通过要求 |
|------|------|----------|
| **P0** | 核心不可用或数据严重错误，阻塞发布 | 必须 100% 通过 |
| **P1** | 主路径受损或高风险逻辑错误 | 必须 100% 通过 |
| **P2** | 非主路径缺陷 | 允许遗留，需评审 |
| **P3** | 文案、轻微视觉、体验类 | 可遗留至下版本 |

---

## 4. Decision（决策格式）

| 决策 | 条件 |
|------|------|
| **PASS** | P0/P1 全部通过，总通过率≥95% |
| **CONDITIONAL** | P0/P1 通过，非核心项未达标，有整改计划 |
| **REJECT** | P0/P1 有未通过项 |

---

```markdown

| 通过率 | XX% |
| P0 通过率 | XX% |
| P1 通过率 | XX% |
| 决策 | PASS / CONDITIONAL / REJECT |
| 未通过项摘要 | ... |
| 发布建议 | 可以发布 / 条件发布 / 暂不发布 |
| 测试负责人 | ___ |
```

---

## 6. Closure（闭环要求）

- REJECT 状态必须跟踪至 P0/P1 全部解决
- CONDITIONAL 需明确整改期限
- 发布决策需 Test Lead + PM 双方确认

---

## 7. Prohibitions（禁止事项）

- 为凑覆盖率编造不可执行用例
- 级别含糊或「全局必须 100%」脱离上下文
- 缺陷描述无法复现

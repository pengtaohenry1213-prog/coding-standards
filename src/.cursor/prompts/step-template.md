# Step 模板（单步任务执行规范）

本模板用于生成标准化的 stepN.md 文件。遵循 CO-STAR 框架的 Mode A 格式。

---

## 1. Context（背景）

Step 是可独立执行的最小单元。一个 step 对应一个独立可验收的任务，符合 README.md"AI 只写 step"的规则。

---

## 2. Objective（目标）

生成标准化的 stepN.md 文件，输出到 `doc/steps/`。

---

## 3. Input（输入）

| 输入 | 说明 |
|------|------|
| 业务需求文档 | PRD、需求描述等 |
| 阶段划分参考 | steps-dev.md 的阶段定义 |

---

## 4. Process（执行流程）

1. 读取业务需求文档
2. 按本模板生成 stepN.md
3. 通过 Human Gate PM 评审

---

## 5. Output（输出物）

### 1. Step 编号

（如：Stage1-Step01，关联大 Plan 的阶段划分）

### 2. Step 目标

（仅聚焦"当前 step 要完成的具体任务"，符合"AI 只写 step"的规则，不写整体代码）

### 3. 输入条件

（完成本 step 需要的前置资源/数据/环境）

### 4. 执行动作

（具体要做的操作，比如"编写 XX 函数""调试 XX 接口""验证 XX 逻辑"）

### 5. 输出物 & 验收标准

（本 step 完成后必须产出的内容 + 验证方式）

| 输出物 | 验收标准 |
|--------|----------|
| | |

### 6. 关联依赖

（依赖的前置 step / 后续 step）

| 依赖类型 | Step | 说明 |
|----------|------|------|
| 前置 | | |
| 后续 | | |

### 7. 测试用例（可选）

| 用例 ID | 测试场景 | 验证方式 |
|----------|----------|----------|
| TC-001 | | |
| TC-002 | | |

### 8. 文件变更清单（可选）

（预估涉及的文件，用于并行执行时的冲突检测）

| 文件 | 操作类型 | 说明 |
|------|----------|------|
| src/xxx.ts | 新增 / 修改 | 功能说明 |

---

## 6. Log Format（日志格式）

```
### HH:MM - [agent: planner]

- 操作：生成 Step
- 文件：doc/steps/step{N}.md
- 执行：读取需求 → 生成 stepN.md
- 结果：完成
```

---

## 7. Git Commit（Git 提交规范）

```
feat(steps): 生成 step{N}
```

---

## 8. Hard Rules（强制规则）

- **step 必须通过 Human Gate PM 评审后才能执行**
- **验收标准必须量化**
- **每个 step 必须有对应的测试用例**
- **拆分原则**：一个 step = 一个独立可验收的单元，工作量 0.5-2 人天

# Human Gate：Security 安全审查

本 prompt 负责对执行过程进行安全审查。遵循 CO-STAR 框架的 Mode B 格式。

---

## 1. Context（背景）

Security Human Gate 是 Human Gate 双审机制的安全关卡，在关键节点触发，确保操作符合安全规范。

### 触发时机

| 节点 | 说明 |
|------|------|
| Gate 1 | Step 执行前（与 PM 联合评审） |
| Gate 2 | Step 执行后（与 PM 联合复审） |
| 敏感操作 | 涉及敏感数据处理时 |
| 危险命令 | 涉及系统命令执行时 |

---

## 2. Objective（审核目标）

确保文件操作、命令执行、密钥管理、数据处理符合安全规范。

---

## 3. Checklist（审核清单）

### 1. 文件操作安全

- [ ] **无路径穿越**：`..` 不出现在路径中
- [ ] **非系统目录**：不操作 `/etc/`, `/root/`, `/sys/`, `/proc/`
- [ ] **非敏感文件**：不覆盖 `.env`, `*.pem`, `credentials.json`, `*.key`
- [ ] **文件大小合理**：无超大文件（异常注入风险）

### 2. 命令执行安全

- [ ] **无危险命令**：
  - `rm -rf`（尤其带通配符）
  - `dd`（磁盘操作）
  - `mkfs`（文件系统创建）
  - `chmod 777`（权限过大）
  - `sudo` 无具体命令
  - 命令修改 `/etc/` 或系统文件
- [ ] **无命令注入**：使用参数数组，非字符串拼接
- [ ] **有超时机制**：防止无限进程

### 3. 密钥与凭证

- [ ] **无硬编码**：密钥/Token 不写在代码中
- [ ] **使用环境变量**：敏感信息从 `process.env` 读取
- [ ] **不输出明文**：日志中敏感字段已脱敏

### 4. 数据安全

- [ ] **敏感字段脱敏**：PII、财务、医疗等数据已掩码
- [ ] **无明文传输**：敏感接口使用 HTTPS
- [ ] **符合等保要求**：如涉及合规数据

---

## 4. Decision（决策格式）

| 决策 | 条件 |
|------|------|
| **PASS** | 所有安全检查项通过 |
| **CONDITIONAL** | 存在非致命风险，需额外注意，需记录处理建议 |
| **REJECT** | 存在高危风险，必须修复后才能继续 |

---

## 5. Decision Record（决策记录）

```markdown
## Human Gate 决策

| 项目 | 内容 |
|------|------|
| Gate 类型 | Security |
| 触发节点 | Gate 1 / Gate 2 / 敏感操作 / 危险命令 |
| 操作类型 | 文件写入 / 命令执行 / API 调用 |
| 操作对象 | ... |
| 决策 | PASS / CONDITIONAL / REJECT |
| 风险描述（如有） | ... |
| 处理建议 | ... |
| 安全负责人 | ___ |
```

---

## 6. Closure（闭环要求）

- REJECT 操作必须终止，等待人工确认
- CONDITIONAL 操作需记录原因，操作后追加日志
- 所有安全事件需写入审计日志

---

## 7. Prohibitions（禁止事项）

```
- 路径穿越
- 系统文件操作
- 危险命令执行
- 硬编码密钥
- 明文敏感数据输出
```
